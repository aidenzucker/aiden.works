MURAL
Collaborative, online whiteboards for connecting global creative teams
Product Design, UX, UI

Product

Murals are online whiteboards where you can collaborate with others in real time, like Google Docs.

Companies like IBM, Intuit and Steelcase use murals to connect their global teams. In 2017, users created ~300K murals and each month 65K+ people use the product.

Role

As one of the five product designers, I worked with PMs, engineering, sales and customer success to improve the experience of using MURAL.

I designed mostly for web, but MURAL works on everything from your iPhone to the 84” inch Microsoft Surface Hub.

Case Studies

Process and outcomes for three projects I led

01

GitHub Integration

Problem

Many teams at IBM use MURAL to breakdown projects into smaller tasks which they put into GitHub for progress tracking. The problem was that engineers were disengaged with the process because it required tedius copy-pasting between MURAL and GitHub.

“I want devs to think ‘Wow, this process is really convenient for building GitHub issues.’ That’s what brings devs to the table.”
—Eric Morrow, Design Facilitator, IBM Cloud

Solution

We designed an integration with GitHub that allows users to quickly create GitHub issues from sticky notes in MURAL with links between the GitHub issue and the sticky note.

1 — Right click group of sticky notes

2 — Select repository and labels

3 — GitHub issues are created

Outcome

In the first half of Q1 2018, 21 users sent 115 sticky notes to GitHub. GitHub recently released an update that will allow IBM to use the integration, so the usage is expected to increase substantially.

02

Educational Loading Screens

Problem

Despite significant performance improvements to MURAL, the most common feedback we got from users continued to be application sluggishness.

We tracked down the negative perception to the loading animation. It was originally designed to be a fun and dynamic, but it ended up looking clunky and associating the MURAL brand with the feeling of indefinite waiting.

We challenged ourselves to make it 10x better.

Before

After

Solution

We illustrated keyboard shortcuts and used emojis to make the messages easy to digest and enjoyable to read. We also found that the perception of loading was lowest when there was a loading bar instead of something spinning perpetually.

Content

The loading tips were written in coordination with the Customer Success team, since they’re the most familiar with the biggest educational needs of MURAL’s users.

03

Unearthing Export

Problem

Through interviews and surveys we learned that users were struggling to get information out of MURAL and into the next stages of their projects.

Hypothesis

There would be less friction getting content out of MURAL if there was an affordance always present in the UI.
